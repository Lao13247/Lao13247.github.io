# Markdown 常用标记

Markdown 是一种轻量级的标记型语言。使用 Markdown 语言创作文章时，可以结合简化的文本标记，进行纯文本格式的文档编写，在发布阶段则会根据不同的语言引擎将标记后的文本内容解析成网页的各种基本元素，简化了排版布局的工作量，更快捷地实现 Html 网页的的内容创作。

# 标题

文章中的标题可以使用 # 符号进行标记， # 接一个空格后再加上文章各标题的文本即可。 # 符号的个数也对应着文章的标题级别，最多允许六级标题，级别越高，标题字号越小。

# 列表

对于要点、说明、步骤等需要列举的内容，在 Markdown 文本可以采用列表标签进行标记。对于并列关系的要点内容等，可以使用无序列表标记，而要更好的表现流程、步骤等内容的顺序结构，可以使用有序列表标记。

无序列表项的标记符号有 - 、 * 、 + ，标记符号加一个空格后再接列表项的文本内容即可，三种符号的标记效果相同，建议使用 - 符号，因为在双语输入法下都能直接输出该字符。另外，次级列表项较比上一级需要在标记前加 4 个空格或者一个 Tab制表符 递进。

有序列表，每项的标记符号直接使用数字和符号 . 进行标序，如 1. 、 2. 等等。

无序列表：

  • 第一项
    • 次第一项
    • 次第二项
  • 第二项
  • 第三项

有序列表：

  1. 第一项
  2. 第二项

# 链接

在 Markdown 文本中可以直接输入网址，将自动转换成链接的形式，样式显示即为网址字符文本，如 www.baidu.com

如网址较长或者不想直接显示网址时，则可以采用 [alt](url "title") 的格式引用链接，其中 alt 项为替代网址显示的内容文本， url 项是需要引用的链接， "title" 为鼠标悬停在链接上的显示文本， title 属性可不配置，配置时不可省略双引号。
对于多个网址的引用还可以使用类似于参考文献的引用格式，即将多个网址罗列在同一处并作好记号，格式为 [alt]:url "title" ，使用时引用对应网址的标记（可以是编号或者文字，支持中文）。

# 图片

Markdown 一般使用图片的网络地址来插入图片，本地图片也可以引用，但是很容易加载失败。
虽然 Markdown 本身支持在文档中插入图片的操作，但是无法对图片的大小、位置等进行控制，影响网页的排版布局，因此建议使用 HTML 标签，结合图床，在文档中插入图片。

其中 【alt】 属性是图片名，也是加载失败时页面替代显示的文本内容， 【url】 则是图片的 url 链接地址， 【title】 属性是鼠标悬浮在图片上时显示的文本内容。

# 引用

创作时，有时候需要引用名人名言、名著佳句等，在 Markdown 中，使用 【>】 符号标记引用，并支持嵌套， 【>】 符号的个数即是此处引用的嵌套层数。

引用效果：

生如夏花之绚烂，死如秋叶之静美。<p align="right">—— 泰戈尔 </p>

  • 注：此处作者部分为让文本右对齐，使用了 HTML 标签的 【align】 属性实现。

# 表格

表格也是常用的数据展示格式，不仅能够直观地展示数据，也能够将数据按照一定的结构组织起来。Markdown 中表格主要有 【单元格】 、 【列名】 和 【分割线】 三种元素：表格中的每个数据都放在单元格中，单元格之间以 【|】 分隔开；列名是表格中第一行的数据内容，是每一列数据的意义说明；在列名与数据单元格之间，即表格的第二行，这行的每个单元格需要固定插入一行分割线 【----】 ，用于文档解析时区分列名与数据单元格。而分割线的左右两端可以添加 【:】 冒号用于该列的文本对齐方式的控制，与 【:】 的位置有关，在分割线左侧即是左对齐，在右侧为右对齐，而两端都有时，则为居中对齐。

# 代码块

Markdown 中，使用一组反引号 【`】 可以实现行内代码的突出显示。而行首的四个连续空格或者 【Tab】 制表符可以标记出一片代码块区域，不过用一组 【```】 可以更直观地标识出代码块区域。

# 分割线

在一行内使用三个或三个以上的 【*】 、 【-】 或者 【_】 都可以在文内画出一条分割线，行内不能有注释等其他内容。

# Markdown 使用技巧

# 支持 HTML 标签

由于 Markdown 文档最终会解析成 HTML 超文本格式，因此，对于 Markdown 原生语法集合无法实现的格式内容，仍然可以在 Markdown 文档中使用 HTML 语言和 CSS 样式来实现和控制。

# 文档注释

文档中有一些说明是用来提示创作者的，并不需要在网页中进行显示。此时，可以就使用文档注释把这部分内容隐藏显示。

  • HTML 注释:
    Markdown 文档支持两种 HTML 注释方法：一种是 HTML 支持的注释语法，即 【<!-- 注释内容 -->】 ；另一种则是将元素的显示属性 【display】 设置为 【none】 ，此时元素的内容即可看作是注释。

# 特殊字符转义

对于 【<】 、 【&】 等特殊字符，使用其字符实体 ( 【&lt】 、 【&amp】 等)，或者使用 【/】 进行转义后才可以直接显示。
而使用一对 【`】 反引号包括直接输入的特殊字符时，则可以强调显示这些特殊字符。反引号 【`】 本身需要使用一对 【```】 才可以强调显示， 【```】 则需要一对 【`】 。

# 文本控制

  • 缩进
    Markdown 文档中，行首连续的四个空格会被识别成代码块标记，而行内的连续空格在解析时也会被被忽略掉。因此，要在 Markdown 文档中实现首行缩进，需要使用代表空格的字符实体如 【&emsp;】 、 【&#8195;】 占位以达到缩进效果。不同格式的空格所占的位置长度不一样，按需选择。而且字符实体后的英文分号 【;】 不可省略。

  • 换行
    在不同的 Markdown 编辑器上，有的使用一次回车也许就可以实现换行，而有道云笔记上需要在上一行末尾添加两个空格才能使用回车换行，VSCode 中需要连续两次回车才能进行换行。

  • 文本对齐
    Markdown 中文本对齐需要使用 HTML 标签的 【align】 属性进行控制。
