个人博客（二）Markdown语法

Markdown是一种轻量级的标记型语言。创作时，允许笔者使用纯文本格式编写文档，而在发布阶段再解析成格式多元、样式丰富的HTML超文本格式用于显示，简化了排版布局的工作量。

一、 Markdown常用标记

Markdown语言通过简化的文本标记，能够更快捷地实现HTML网页的各种基本元素的内容创作。标题、列表等行元素的标记与内容之间至少有一个空格才能正常显示。

1. 标题

标题使用【#】符号进行标记。符号的个数对应标题级别，最多不超过六级。字号逐渐减小，一级标题字号最大，六级标题字号最小。

# 一级标题
## 二级标题
### 三级标题
#### 四级标题
##### 五级标题
###### 六级标题

2. 列表

  • 无序列表
    【-】、【*】、【+】都可以作为无序列表的项标记，效果相同。【-】输入法的双语下都能够直接输入，因此比较常用。

    * 第一项
    * 次第一项
+ 第二项 
- 第三项

  • 有序列表
    使用1.、2.编号标序即可实现有序列表，

1. 第一项
    1.1 次第一项
2. 第二项

3. 字体

Markdown可以使用标记来使用加粗、斜体等常用字体。

*斜体*或_斜体_
**粗体**
***加粗斜体***
~~删除线~~
++下划线++
==背景高亮==

4. 链接

​ Markdown中引用链接有多种方法：直接输入网址，就可以自动转换成链接的形式，此时链接显示的文本为网址本身；还可以使用【[alt](url "title")】的格式引用链接，适用于网址较长的情况，此时链接的显示文本为【alt】的内容，【url】则是引用的网址，【"title"】是鼠标悬停在链接上时显示的文本内容，不是必须使用的属性，一般会省略。
多个网址的引用还可以使用类似于参考文献的引用格式，即将多个网址罗列在同一处并作好记号，格式为【[alt]:url "title"】，使用时引用对应网址的标记（可以是编号或者文字，支持中文）。
链接使用示例：

①直接输入网址：www.baidu.com
②使用[站名](网址)格式：[百度](www.baidu.com)
③也可以通过脚注的方式隐藏链接，格式为[站名][脚注名][Google][Google]、[简书][简书]。

[Google]:http://www.google.com "Google"
[简书]:http://www.jianshu.com "简书"

链接语法效果（按住【Ctrl】+ 鼠标左键点击链接即可打开网址）：
①直接输入网址：www.baidu.com
②使用【[alt](url)】的格式：百度
③我经常去的几个网站Google、简书。

5. 图片

Markdown一般使用图片的网络地址来插入图片，本地图片也可以引用，但是很容易加载失败。
虽然Markdown本身支持在文档中插入图片的操作，但是无法对图片的大小、位置等进行控制，影响网页的排版布局，因此建议使用HTML标签，结合图床，在文档中插入图片。

①Markdown插入图片：
![alt](图片url "title")

②HTML插入图片：
<img src=图片url witdh="" height="" title="" alt=""/>

其中【alt】属性是图片名，也是加载失败时页面替代显示的文本内容，【url】则是图片的url链接地址，【title】属性是鼠标悬浮在图片上时显示的文本内容。

6. 引用

创作时，有时候需要引用名人名言、名著佳句等，在Markdown中，使用【>】符号标记引用，并支持嵌套，【>】符号的个数即是此处引用的嵌套层数。

> 生如夏花之绚烂，死如秋叶之静美。<p align="right">——泰戈尔</p>

引用效果：

生如夏花之绚烂，死如秋叶之静美。

——泰戈尔

  • 注：此处作者部分为让文本右对齐，使用了HTML标签的【align】属性实现。

7. 表格

表格也是常用的数据展示格式，不仅能够直观地展示数据，也能够将数据按照一定的结构组织起来。Markdown中表格主要有【单元格】、【列名】和【分割线】三种元素：表格中的每个数据都放在单元格中，单元格之间以【|】分隔开；列名是表格中第一行的数据内容，是每一列数据的意义说明；在列名与数据单元格之间，即表格的第二行，这行的每个单元格需要固定插入一行分割线【----】，用于文档解析时区分列名与数据单元格。而分割线的左右两端可以添加【:】冒号用于该列的文本对齐方式的控制，与【:】的位置有关，在分割线左侧即是左对齐，在右侧为右对齐，而两端都有时，则为居中对齐。
表格使用示例：

<!-- 列名 -->
<!-- 分割线/文本对齐控制 -->
<!-- 数据单元格 -->
| 左对齐 | 右对齐 | 居中对齐 |
| :---- | ----: | :----: |
| 单元格 | 单元格 | 单元格 |
| 单元格 | 单元格 | 单元格 |

效果：

左对齐右对齐居中对齐
单元格单元格单元格
单元格单元格单元格
  • 注意：不能在表格内容的所在行编写注释，否则会破坏表格格式，无法正常显示。

7. 代码块

Markdown中，使用一组反引号【`】可以实现行内代码的突出显示。而行首的四个连续空格或者【Tab】制表符可以标记出一片代码块区域，不过用一组【```】可以更直观地标识出代码块区域。

#include <stdio.h>

int main(int argc, char *argv[]) {
    printf("Hello C World!");
    return 0; 
}

8. 分割线

在一行内使用三个或三个以上的【*】、【-】或者【_】都可以在文内画出一条分割线，行内不能有注释等其他内容。

二、 Markdown使用技巧

1. 支持HTML标签

由于Markdown文档最终会解析成HTML超文本格式，因此，对于Markdown原生语法集合无法实现的格式内容，仍然可以在Markdown文档中使用HTML语言和CSS样式来实现和控制。

2. 文档注释

文档中有一些说明是用来提示创作者的，并不需要在网页中进行显示。此时，可以就使用文档注释把这部分内容隐藏显示。

  • HTML注释:
    Markdown文档支持两种HTML注释方法：一种是HTML支持的注释语法，即【<!-- 注释内容 -->】；另一种则是将元素的显示属性【display】设置为【none】，此时元素的内容即可看作是注释。

    <!-- 我是HTML直接支持的注释语法，单行注释  -->

<div style='display: none'>
    我是HTML元素中的注释内容
</ div>

3. 特殊字符转义

对于【<】、【&】等特殊字符，使用其字符实体(【&lt】、【&amp】等)，或者使用【/】进行转义后才可以直接显示。
而使用一对【`】 反引号包括直接输入的特殊字符时，则可以强调显示这些特殊字符。反引号【`】本身需要使用一对【```】才可以强调显示,【```】则需要一对【`】。

4. 文本控制

  • 缩进
    Markdown文档中，行首连续的四个空格会被识别成代码块标记，而行内的连续空格在解析时也会被被忽略掉。因此，要在Markdown文档中实现首行缩进，需要使用代表空格的字符实体如【&emsp;】、【&#8195;】占位以达到缩进效果。不同格式的空格所占的位置长度不一样，按需选择。而且字符实体后的英文分号【;】不可省略。
1. &emsp;或&#8195; //全角
2. &ensp;或&#8194; //半角
3. &nbsp;或&#160;  //半角的一半

  • 换行
    在不同的Markdown编辑器上，有的使用一次回车也许就可以实现换行，而有道云笔记上需要在上一行末尾添加两个空格才能使用回车换行，VSCode中需要连续两次回车才能进行换行。

  • 文本对齐
    Markdown中文本对齐需要使用HTML标签的【align】属性进行控制。

<center>行居中对齐</center>
<div align="center">行居中对齐</div>
<p align="left">行左对齐</p>
<p align="right">行右对齐</p>

三、 Markdown进阶使用

Markdown借助插件、工具等还可以实现数学公式编辑、流程图绘制等效果，不过此处暂不表述。
