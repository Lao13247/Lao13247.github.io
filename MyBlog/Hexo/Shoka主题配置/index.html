一、Shoka主题简介

Shoka主题是主题作者参考笔记软件Bear和Hexo博客主题Next设计的一款样式标签丰富、功能齐全的二次元风格主题，支持分类精选、文章配图和多媒体播放等功能，非常适合作为笔记记录和博客分享的展示。

二、下载主题

在博客根目录下右键打开Git Bash Here，输入以下命令克隆安装主题包：

git clone https://github.com/amehime/hexo-theme-shoka.git ./themes/shoka

命令执行成功结束后，博客根目录\themes\目录下有shoka文件夹即安装成功。

三、安装依赖插件

Hexo博客的许多功能和样式效果通常都是由各种插件提供支持的，通过npm命令安装以下Shoka主题所需功能插件：

插件名称安装命令功能影响
hexo-autoprefixernpm i hexo-autoprefixer –save给生成的css添加浏览器前缀必须安装的插件
hexo-algoliasearchnpm i hexo-algoliasearch –save站内搜索搜索按钮失灵
hexo-symbols-count-timenpm i hexo-symbols-count-time –save文章、站点字数及阅读时间统计无统计数据
hexo-feednpm i hexo-feed –save生成Feed文件无Feed文件

另外，Shoka主题采用的markdown渲染引擎基于markdown-it，需要卸载Hexo框架自带的hexo-renderer-marked渲染引擎或者其他引擎。

卸载hexo-renderer-marked默认引擎：

npm un hexo-renderer-marked --save

安装hexo-renderer-multi-markdown-it渲染引擎：

npm i hexo-renderer-multi-markdown-it --save --ignore-scripts

此处命令在引擎安装时跳过了puppeteer组件的Chromium内核下载，这样做是为了避免国内网络无法下载Chromium内核导致引擎安装失败的情况。但是跳过内核下载将会影响mermaid流程图标签功能的使用，所以网络条件能下载时尽量下载，去除命令末尾的--ignore-scripts命令参数即可。

三、站点配置

在博客根目录以及主题目录下，均有一个名称为_config.yml的配置文件，根目录下的为全局配置文件，主要用于配置站点基础信息以及控制主题插件效果等。主题目录下的则是主题配置文件，保存的是主题特色样式或功能的配置：

全局配置文件路径：博客根目录\_config.yml。

主题配置文件路径：博客根目录\themes\shoka\_config.yml。

也可以在博客根目录下创建_config.shoka.yml文件保存自定义的主题配置，方便主题升级时保留自定义配置。

主题和插件安装好后，就可以开始进行站点的配置了。

（一）全局配置文件调整

以下所有配置均在全局配置文件中调整或添加：

  1. 修改主题。Hexo博客默认主题为landscape，需要修改为shoka

    theme: shoka

  2. 渲染引擎配置。加入markdown配置，使安装的markdown-it渲染引擎生效

    markdown:
  render: # 渲染器设置
    html: false # 过滤 HTML 标签
    xhtmlOut: true # 使用 '/' 来闭合单标签 （比如 <br />）。
    breaks: true # 转换段落里的 '\n' 到 <br>。
    linkify: true # 将类似 URL 的文本自动转换为链接。
    typographer:
    quotes: '“”‘’'
  plugins: # markdown-it插件设置
    - plugin:
        name: markdown-it-toc-and-anchor
        enable: true
        options: # 文章目录以及锚点应用的class名称，shoka主题必须设置成这样
          tocClassName: 'toc'
          anchorClassName: 'anchor'
    - plugin:
        name: markdown-it-multimd-table
        enable: true
        options:
          multiline: true
          rowspan: true
          headerless: true
    - plugin:
        name: ./markdown-it-furigana
        enable: true
        options:
          fallbackParens: "()"
    - plugin:
        name: ./markdown-it-spoiler
        enable: true
        options:
          title: "你知道得太多了"

  3. 加入minify配置，用于压缩文件，优化网页渲染速度

    minify:
  html:
    enable: true
    exclude: # 排除 hexo-feed 用到的模板文件
      - '**/json.ejs'
      - '**/atom.ejs'
      - '**/rss.ejs'
  css:
    enable: true
    exclude:
      - '**/*.min.css'
  js:
    enable: true
    mangle:
      toplevel: true
    output:
    compress:
    exclude:
      - '**/*.min.js'

  4. 停用默认代码高亮功能。Shoka主题自带了Mac样式的代码高亮功能，只有停用默认代码高亮功能才能正常显示。

    修改highlight和prismjs配置，将enable项的配置值均改为false即可

    highlight:
  enable: false
  
prismjs:
  enable: false

  5. 加入autoprefixer配置

    autoprefixer:
  exclude:
    - '*.min.css'

    缺少这个插件配置会影响首页的卡片翻转效果在部分浏览器的显示。

  6. 加入algolia配置，配置搜索功能

    # algolia:
#   appId:
#   apiKey:
#   adminApiKey:
#   chunkSize: 5000
#   indexName:
#   fields:
#     - title #必须配置
#     - path #必须配置
#     - categories #推荐配置
#     - content:strip:truncate,0,4000
#     - gallery
#     - photos
#     - tags

    此处主题默认配置注释掉了，可能需要自行注册algolia账号进行相关配置后再启用。

  7. 加入feed配置，用于生成feed文件

    feed:
    limit: 20
    order_by: "-date"
    tag_dir: false
    category_dir: false
    rss:
        enable: true
        template: "themes/shoka/layout/_alternate/rss.ejs"
        output: "rss.xml"
    atom:
        enable: true
        template: "themes/shoka/layout/_alternate/atom.ejs"
        output: "atom.xml"
    jsonFeed:
        enable: true
        template: "themes/shoka/layout/_alternate/json.ejs"
        output: "feed.json"

（二）主题配置文件调整

下列配置均在主题配置文件中调整或添加：

  1. 配置站点别称。alternate配置的名称将代替Logo，显示在页面顶部和底部版权标识©处。

    alternate: YU NIAN

  2. 渲染资源的静态目录配置如下：

    statics: / #//cdn.jsdelivr.net/gh/username/RepositoryName@latest/
css: css
js: js
images: images

    statics项值为/表示使用站点本地的静态资源，也可以使用jsDelivr加速线上文件，配置格式为//cdn.jsdelivr.net/gh/github用户名/仓库名称@latest/。其他项是对应格式的静态资源的加载路径，一般不修改。

  3. Shoka主题可以在加载和切换页面时添加过渡动画，按需求开启页面首进或页面切换的动画效果。

    loader:
  start: true # When entering the page
  switch: false # When switching to another page

  4. 自动定位指再次打开页面时，页面滚轮自动定位到上次浏览的位置。Shoka主题默认开启自动定位功能，配置设为false时将停用该功能。

    auto_scroll: true

  5. Shoka主题鼠标单击页面时会有烟花效果，启用时配置如下，可以自行调整。

    fireworks:
  enable: true # 是否启用
  color: # 烟花颜色
    - "rgba(255,182,185,.9)"
    - "rgba(250,227,217,.9)"
    - "rgba(187,222,214,.9)"
    - "rgba(138,198,209,.9)"

  6. Shoka主题为了友好显示日文等字体，使用了谷歌字体。

    font:
  enable: true
  # Font options:
  # `external: true` will load this font family from `host` above.
  # `family: Times New Roman`. Without any quotes.
  # `size: x.x`. Use `em` as unit. Default: 1 (16px)

  # Global font settings used for all elements inside <body>.
  global:
    external: true
    family: Mulish
    size:

  # Font settings for alternate title.
  logo:
    external: true
    family: Fredericka the Great
    size: 3.5

  # Font settings for site title.
  title:
    external: true
    family: Noto Serif JP
    size: 2.5

  # Font settings for headlines (<h1> to <h6>).
  headings:
    external: true
    family: Noto Serif SC
    size:

  # Font settings for posts.
  posts:
    external: true
    family:

  # Font settings for <code> and code blocks.
  codes:
    external: true
    family: Inconsolata

  7. iconfont图标

    Shoka主题对Font Awesome图标进行了精简和修改，Shoka图标清单保存在博客根目录/themes/shoka/source/css/_iconfont.styl文件中。

    # project of https://www.iconfont.cn/
# //at.alicdn.com/t/font_1832207_c8i9n1ulxlt.css => 1832207_c8i9n1ulxlt
iconfont: "1832207_c8i9n1ulxlt"

    如果需要添加或修改图标样式，需要与Shoka主题拥有者联系，具体见Shoka主题基础配置中的iconfont配置部分。

  8. 菜单与社交按钮配置，用于调整页面顶部的菜单导航以及简介页面上展示的个人其他平台的链接跳转和图标样式。

    menu:
  home: / || home
  about: /about/ || user
  posts:
    default: / || feather
    archives: /archives/ || list-alt
    categories: /categories/ || th
    tags: /tags/ || tags
    friends: /friends/ || heart
    links: /links/ || magic
social:
  github: https://github.com/yourname || github || "#191717"
  #google: https://plus.google.com/yourname || google
  #twitter: https://twitter.com/yourname || twitter || "#00aff0"
  #zhihu: https://www.zhihu.com/people/yourname || zhihu || "#1e88e5"
  #music: https://music.163.com/#/user/home?id=yourid || cloud-music || "#e60026"
  #weibo: https://weibo.com/yourname || weibo || "#ea716e"
  #about: https://about.me/yourname || address-card || "#3b5998"
  #email: mailto:yourname@mail.com || envelope || "#55acd5"
  #facebook: https://www.facebook.com/yourname || facebook
  #stackoverflow: https://stackoverflow.com/yourname || stack-overflow
  #youtube: https://youtube.com/yourname || youtube
  #instagram: https://instagram.com/yourname || instagram
  #skype: skype:yourname?call|chat || skype
  #douban: https://www.douban.com/people/yourname/ || douban

    各项的配置值格式为链接或路径 || 图标名称 || 颜色值。其中：

    • ||作为分隔符，前后均有一个空格
    • 图标配置时只需要主体名称，不用添加前缀，如图标配置为github，解析时则会自动转换格式为ic i-github
    • 十六进制的颜色码需要使用英文双引号""包括起来

另外，菜单menu各项支持设置一级子目录，但子目录的第一项必须为default，用于指定父级按钮的样式。每个按钮显示的文字可以在语言包中调整，参考菜单文字显示。

  1. 侧边栏用于展示个人头像、简介以及平台链接等信息，配置sidebar区域：

    sidebar:
  # Sidebar Position.
  position: left
  #position: right
  # Replace the default avatar image and set the url here.
  avatar: avatar.jpg

    position项配置侧边栏展示在左侧或者右侧；avatar项是设置个人头像，头像图片需要放在博客根目录/source/_data/images/目录下，也可以配置在线图片。

  2. Shoka主题在页面底部可以配置展示随机文章和最近评论两个区域部件。

    widgets:
  random_posts: true # 显示随机文章
  recent_comments: true # 显示最近评论

  3. 字数和阅读时间统计配置。统计数据需要安装hexo-symbols-count-time插件支持。在全局配置文件中保持默认插件配置，在主题配置文件中，修改footer和post`配置：

    # 页尾全站统计
footer:
  since: 2023
  count: true
# 文章界面统计
post:
  count: true

  4. Hexo内置MiniValine文章评论系统。使用前需要在Valine官网注册获取自己的appId和appKey，参考Valine官网教程。

    除了配置appId和appKey之外，可以根据需要配置评论者名称后的tag标签属性tagMeta、tagColor和tagMember等，默认访客角色为visitor。

    访客角色visitor的键名不能修改，但是visitor的键值或其他key键的名称与值均可以替换，角色key键名应当统一对应。

    valine:
  appId: #Your_appId
  appKey: #Your_appkey
  placeholder: ヽ(○´∀`)ﾉ♪ # Comment box placeholder
  avatar: mp # Gravatar style : mp, identicon, monsterid, wavatar, robohash, retro
  pageSize: 10 # Pagination size
  lang: zh-CN
  visitor: true # 文章访问量统计
  NoRecordIP: false # 不记录 IP
  serverURLs: # When the custom domain name is enabled, fill it in here (it will be detected automatically by default, no need to fill in)
  powerMode: true # 默认打开评论框输入特效
  tagMeta:
    visitor: 新朋友
    master: 主人
    friend: 小伙伴
    investor: 金主粑粑
  tagColor:
    master: "var(--color-orange)"
    friend: "var(--color-aqua)"
    investor: "var(--color-pink)"
  tagMember:
    master:
      # - hash of master@email.com
      # - hash of master2@email.com
    friend:
      # - hash of friend@email.com
      # - hash of friend2@email.com
    investor:
      # - hash of investor1@email.com

    在文章头部参数区Front Matter部分也可以单独配置评论系统设置，比如评论提示placeholder项。

    ---
valine:
  placeholder: 评论提示语
---

    如果某些文章要单独关闭评论系统，可以在头部参数中配置comment参数为false:

    ---
title: 文章标题
comment: false
---

  5. 点击页面顶部右侧的阳光样式的按钮即可切换博客的日间或夜间两种背景模式，需要先在主题配置中开启夜间模式。

    darkmode: true

    默认情况下，博客背景模式取决于（优先级从高到低）：切换按钮选择；切换设备浏览器色调；darkmode 配置项。

  7. 文章卡片的随机图片来源是新浪图库，图片列表在博客根目录\themes\shoka\_images.yml文件中，可以自行上传到新浪图库后，在图片列表中添加新一行的外链图片名称及后缀，或者图片URL。

    - 6833939bly1gicmnywqgpj20zk0m8dwx.jpg
- https://i.loli.net/2020/10/30/rjdhcSgEN8COBPA.jpg

    若将已有图床作为随机图库，可以在主题配置文件中添加图床API配置。

    image_server: "https://acg.xydwz.cn/api/api.php"

  8. 主题配置文件中的vendors配置用于加载第三方组件，文件加载基于jsDelivr加速，并进行了文件合并，所以不了解时不要轻易修改。

(三)阅读相关配置

  1. 文章置顶。在文章头部参数中配置sticky: true可将文章显示在博客首页的置顶区，多篇文章置顶则按照发布时间倒序排列，显示不分页。

    ---
sticky: true
---

  2. 文章分类卡片。文章分类想要在首页卡片形式显示，需要在博客对应分类的目录下存放封面图cover.jpg，封面图文件名称不可修改。

    在全局配置文件中的catogary_map配置项里配置文件名与文章链接显示值的对应关系，避免文章链接中出现中文。

    category_map: 
  个人博客: blog

    另外，文章目录侧的系列文章显示的是处于同一分类下的文章，按照文章名称正序排列。

  3. 在文章头部参数区配置cover项及图片路径可以指定文章在首页显示的封面图片，封面图片的相对路径应在source或其子目录下，也可以配置图片URL采用外链图片，不配置则随机显示一张随机图库中的图片。若在头部参数区配置了photos项引用多张图片而未配置cover项时，文章封面则会显示photos项的第一张图片。

    ---
cover: assets/cover-1.jpg
---

  4. 文章页面也可以展示图片及相册，即在头部参数区配置前面说到的photos项。

    photos:
  - assets/photo-1.jpg
  - assets/photo-2.jpg
  - https://api.com/photo-3.jpg

    或者在文章内容的标题前面用markdown语法引用多张图片，再用Justified Gallery对图片进行排列。

    ![](https://tva3.sinaimg.cn/large/6833939bly1giclfdu6exj20zk0m87hw.jpg)
![](https://tva3.sinaimg.cn/large/6833939bly1giclflwv2aj20zk0m84qp.jpg)
![](https://tva3.sinaimg.cn/large/6833939bly1giclg5ms2rj20zk0m8u0x.jpg)
![](https://tva3.sinaimg.cn/large/6833939bly1giclhnx9glj20zk0m8npd.jpg)
{.gallery data-height="120"}

    其中data-height设置图片行的高度，默认值为220

  5. 主题配色。页面配色文件路径为博客根目录\themes\shoka\source\css\_colors.styl，自定义页面配色可以在博客根目录\source\_data目录下新建colors.styl文件，复制粘贴主题页面配色文件内容后，在新建页面配色文件中添改配色，为避免样式显示异常，原有样式名称均要存在。同理，图标配色文件也可以如是调整，图标配色的自定义文件名为iconfont.styl，如要添加自定义样式，可以在目录中创建custom.styl文件进行配置。

  6. 主题图片替换。博客根目录\themes\shoka\source\images\存放着头像、打赏图片等主题相关图片资源，如要替换图片，可以在博客根目录\source\_data目录下新建images目录，将替换的图片名称修改为与对应主题图片同名后放到此处即可覆盖默认主题图片。

  7. 自定义语言包用于替换menu菜单按钮或者默认区域的文字显示，默认语言包在博客根目录\themes\shoka\languages\下。

  3. quiz练习题。在文章头部参数区配置quiz: true可以设置练习题。使用{.quiz}等标签标注题型及选项答案。功能基于markdown-it-bracketed-spans和markdown-it-attrs。

    标签说明
    {.quiz}选择题
    {.quiz .multi}多选题
    {.quiz .true}正确的判断题
    {.quiz .false}错误的判断题
    {.quiz .fill}填空题
    []{.gap}空白下划线
    [答案内容]{.gap}答案内容带下划线
    {.options}ABCDE 选项
    {.correct}选择题的正确选项
    >答案解析
    [8.4]{.mistake}错题备注
    1. 编译时多态主要指运算符重载与函数重载，而运行时多态主要指虚函数。 {.quiz .true}

2. 有基类`SHAPE`，派生类`CIRCLE`，声明如下变量：  {.quiz .multi}
    ```cpp
    SHAPE shape1,*p1;
    CIRCLE circle1,*q1;
    ```
    下列哪些项是“派生类对象替换基类对象”。
    - `p1=&circle1;` {.correct}
    - `q1=&shape1;`
    - `shape1=circle1;` {.correct}
    - `circle1=shape1;`
 {.options}
    > - :heavy_check_mark: 令基类对象的指针指向派生类对象
    > - :x: 派生类指针指向基类的引用
    > - :heavy_check_mark: 派生类对象给基类对象赋值
    > - :x: 基类对象给派生类对象赋值
    > {.options}

3. 下列叙述正确的是 []{.gap} 。 {.quiz}
    - 虚函数只能定义成无参函数
    - 虚函数不能有返回值
    - 能定义虚构造函数
    - A、B、C都不对 {.correct}
{.options}

4. 如果定义`int e=8; double f=6.4, g=8.9;`，则表达式 `f+int(e/3*int(f+g)/2)%4` 的值为 [9.4]{.gap}。 {.quiz .fill}
    > 注意运算顺序和数据类型
    > [8.4]{.mistake}
