# 一、Shoka 主题简介

Shoka 主题是主题作者参考笔记软件 Bear 和 Hexo 博客主题 Next 设计的一款样式标签丰富、功能齐全的二次元风格主题，支持分类精选、文章配图和多媒体播放等功能，非常适合作为笔记记录和博客分享的展示。

# 三、安装依赖插件

Hexo 博客的许多功能和样式效果通常都是由各种插件提供支持的，通过 npm 命令安装以下 Shoka 主题所需功能插件：

插件名称安装命令功能影响
hexo-autoprefixernpm i hexo-autoprefixer --save给生成的 css 添加浏览器前缀必须安装的插件
hexo-algoliasearchnpm i hexo-algoliasearch --save站内搜索搜索按钮失灵
hexo-symbols-count-timenpm i hexo-symbols-count-time --save文章、站点字数及阅读时间统计无统计数据
hexo-feednpm i hexo-feed --save生成 Feed 文件无 Feed 文件

此处命令在引擎安装时跳过了 puppeteer 组件的 Chromium内核 下载，这样做是为了避免国内网络无法下载 Chromium内核 导致引擎安装失败的情况。但是跳过内核下载将会影响 mermaid流程图 标签功能的使用，所以网络条件能下载时尽量下载，去除命令末尾的 --ignore-scripts 命令参数即可。

# 三、站点配置

在博客根目录以及主题目录下，均有一个名称为 _config.yml 的配置文件，根目录下的为 全局配置文件 ，主要用于配置站点基础信息以及控制主题插件效果等。主题目录下的则是 主题配置文件 ，保存的是主题特色样式或功能的配置：

全局配置文件 路径： 博客根目录\_config.yml 。

主题配置文件 路径： 博客根目录\themes\shoka\_config.yml 。

也可以在博客根目录下创建 _config.shoka.yml 文件保存自定义的主题配置，方便主题升级时保留自定义配置。

主题和插件安装好后，就可以开始进行站点的配置了。

# （一）全局配置文件调整

以下所有配置均在全局配置文件中调整或添加：

  1. 修改主题。Hexo 博客默认主题为 landscape ，需要修改为 shoka

  2. 渲染引擎配置。加入 markdown 配置，使安装的 markdown-it 渲染引擎生效

  3. 加入 minify 配置，用于压缩文件，优化网页渲染速度

  4. 停用默认代码高亮功能。Shoka 主题自带了 Mac 样式的代码高亮功能，只有停用默认代码高亮功能才能正常显示。

    修改 highlight 和 prismjs 配置，将 enable 项的配置值均改为 false 即可

  5. 加入 autoprefixer 配置

    缺少这个插件配置会影响首页的卡片翻转效果在部分浏览器的显示。

  6. 加入 algolia 配置，配置搜索功能

    此处主题默认配置注释掉了，可能需要自行注册 algolia 账号进行相关配置后再启用。

  7. 加入 feed 配置，用于生成 feed 文件

# （二）主题配置文件调整

下列配置均在主题配置文件中调整或添加：

  1. 配置站点别称。 alternate 配置的名称将代替 Logo，显示在页面顶部和底部版权标识 © 处。

  2. 渲染资源的静态目录配置如下：

    statics 项值为 / 表示使用站点本地的静态资源，也可以使用 jsDelivr 加速线上文件，配置格式为 //cdn.jsdelivr.net/gh/github用户名/仓库名称@latest/ 。其他项是对应格式的静态资源的加载路径，一般不修改。

  3. Shoka 主题可以在加载和切换页面时添加过渡动画，按需求开启页面首进或页面切换的动画效果。

  4. 自动定位指再次打开页面时，页面滚轮自动定位到上次浏览的位置。Shoka 主题默认开启自动定位功能，配置设为 false 时将停用该功能。

  5. Shoka 主题鼠标单击页面时会有烟花效果，启用时配置如下，可以自行调整。

  6. Shoka 主题为了友好显示日文等字体，使用了谷歌字体。

  7. iconfont 图标

    Shoka 主题对 Font Awesome 图标进行了精简和修改，Shoka 图标清单保存在 博客根目录/themes/shoka/source/css/_iconfont.styl 文件中。

    如果需要添加或修改图标样式，需要与 Shoka 主题拥有者联系，具体见 Shoka 主题基础配置中的 iconfont配置 部分。

  8. 菜单与社交按钮配置，用于调整页面顶部的菜单导航以及简介页面上展示的个人其他平台的链接跳转和图标样式。

    各项的配置值格式为 链接或路径 || 图标名称 || 颜色值 。其中：

    • || 作为分隔符，前后均有一个空格
    • 图标配置时只需要主体名称，不用添加前缀，如图标配置为 github ，解析时则会自动转换格式为 ic i-github
    • 十六进制的颜色码需要使用英文双引号 "" 包括起来

另外，菜单 menu 各项支持设置一级子目录，但子目录的第一项必须为 default ，用于指定父级按钮的样式。每个按钮显示的文字可以在语言包中调整，参考菜单文字显示。

  1. 侧边栏用于展示个人头像、简介以及平台链接等信息，配置 sidebar 区域：

    position 项配置侧边栏展示在左侧或者右侧； avatar 项是设置个人头像，头像图片需要放在 博客根目录/source/_data/images/ 目录下，也可以配置在线图片。

  2. Shoka 主题在页面底部可以配置展示 随机文章 和 最近评论 两个区域部件。

  3. 字数和阅读时间统计配置。统计数据需要安装 hexo-symbols-count-time 插件支持。在 全局配置文件 中保持默认插件配置，在主题配置文件 中，修改 footer 和 post` 配置：

  4. Hexo 内置 MiniValine 文章评论系统。使用前需要在 Valine 官网注册获取自己的 appId 和 appKey ，参考 Valine 官网教程。

    除了配置 appId 和 appKey 之外，可以根据需要配置评论者名称后的 tag 标签属性 tagMeta 、 tagColor 和 tagMember 等，默认访客角色为 visitor 。

    访客角色 visitor 的键名不能修改，但是 visitor 的键值或其他 key 键的名称与值均可以替换，角色 key 键名应当统一对应。

    在文章头部参数区 Front Matter 部分也可以单独配置评论系统设置，比如评论提示 placeholder 项。

    如果某些文章要单独关闭评论系统，可以在头部参数中配置 comment 参数为 false :

  5. 点击页面顶部右侧的阳光样式的按钮即可切换博客的日间或夜间两种背景模式，需要先在主题配置中开启夜间模式。

    默认情况下，博客背景模式取决于（优先级从高到低）：切换按钮选择；切换设备浏览器色调；darkmode 配置项。

  6. Shoka 主题可以直接使用网易云、虾米、QQ 音乐等平台的单曲或歌单的 URL 链接作为背景音乐播放列表。可以在 主题配置文件 中设置 全局播放列表 ，也可以在文章的 头部参数区 单独设置播放列表。

    也可以播放自定义媒体文件。

    如果文章需要关闭背景音乐，在文章头部参数区配置 audio 参数。

  7. 文章卡片的随机图片来源是新浪图库，图片列表在 博客根目录\themes\shoka\_images.yml 文件中，可以自行上传到新浪图库后，在图片列表中添加新一行的 外链图片名称及后缀 ，或者 图片URL 。

    也可以自定义图片列表，在 博客根目录\source\_data 目录下新建 images.yml 文件，补充六张以上的图片作为自定义列表，将完全覆盖原来的主题图片列表。

    若将已有图床作为随机图库，可以在主题配置文件中添加 图床API 配置。

  8. 主题配置文件中的 vendors 配置用于加载第三方组件，文件加载基于 jsDelivr 加速，并进行了文件合并，所以不了解时不要轻易修改。

    组件名称组件功能作用域
    pace加载进度条全局
    pjax页面无刷新加载全局
    animejs 动画效果全局
    algolia,instantsearch基于 algolia 的站内搜索全局
    lazyload图片懒加载全局
    quicklink链接资源预加载全局
    fetch获取播放列表全局
    katex copy_tex数学公式显示及复制按需
    fancybox图片放大显示及排列按需
    valine基于 LeanCloud 的评论系统及文章阅读次数统计按需
    chart图表显示按需

# (三) 阅读相关配置

  1. 文章置顶。在文章头部参数中配置 sticky: true 可将文章显示在博客首页的置顶区，多篇文章置顶则按照发布时间倒序排列，显示不分页。

  2. 文章分类卡片。文章分类想要在首页卡片形式显示，需要在博客对应分类的目录下存放封面图 cover.jpg ，封面图文件名称不可修改。

    在全局配置文件中的 catogary_map 配置项里配置文件名与文章链接显示值的对应关系，避免文章链接中出现中文。

    另外， 文章目录 侧的 系列文章 显示的是处于同一分类下的文章，按照文章名称正序排列。

  3. 在文章头部参数区配置 cover 项及图片路径可以指定文章在首页显示的封面图片，封面图片的相对路径应在 source 或其子目录下，也可以配置图片 URL 采用外链图片，不配置则随机显示一张随机图库中的图片。若在头部参数区配置了 photos 项引用多张图片而未配置 cover 项时，文章封面则会显示 photos 项的第一张图片。

  4. 文章页面也可以展示图片及相册，即在头部参数区配置前面说到的 photos 项。

    或者在文章内容的标题前面用 markdown 语法引用多张图片，再用 Justified Gallery 对图片进行排列。

    其中 data-height 设置图片行的高度，默认值为 220

  5. 主题配色。页面配色文件路径为 博客根目录\themes\shoka\source\css\_colors.styl ，自定义页面配色可以在 博客根目录\source\_data 目录下新建 colors.styl 文件，复制粘贴主题页面配色文件内容后，在新建页面配色文件中添改配色，为避免样式显示异常，原有样式名称均要存在。同理，图标配色文件也可以如是调整，图标配色的自定义文件名为 iconfont.styl ，如要添加自定义样式，可以在目录中创建 custom.styl 文件进行配置。

  6. 主题图片替换。 博客根目录\themes\shoka\source\images\ 存放着头像、打赏图片等主题相关图片资源，如要替换图片，可以在 博客根目录\source\_data 目录下新建 images 目录，将替换的图片名称修改为与对应主题图片同名后放到此处即可覆盖默认主题图片。

  7. 自定义语言包用于替换 menu菜单按钮 或者默认区域的文字显示，默认语言包在 博客根目录\themes\shoka\languages\ 下。

# 四、特殊语法

  1. 链接块。将链接字段信息写在 links 标签块内，可以将链接转换成标签卡片的形式显示，便于排列和美观。可配置链接字段信息见下表：

    字段字段含义配置
    site站点名称必填
    owner管理员名字可选，默认为 site 的值
    url站点链接必填
    desc站点描述可选，默认为 url 的值
    image站点图片可选，默认为 images/404.png
    color方块颜色可选，默认为 #666

    優萌初華

    琉璃的医学 & 编程笔记

    優萌初華

    https://shoka.lostyu.me

    優萌初華

    琉璃的医学 & 编程笔记

    在文章中配置大量链接会影响篇幅，所以可以在 source 或其子目录下用 yml文件 保存所有的链接信息，在文章中用 linksfile 标签引用 yml 文件中的站点信息。

  2. 代码块。代码块样式类似 mac 命令行，高亮功能采用 prism 组件，代码块信息格式： [language] [title] [url] [link text] [mark] [command]

    字段字段描述默认值
    language如不需要代码高亮，但希望显示代码块样式，则设为 raw，留空或设为 info ，将不显示代码高亮和代码块样式null
    title代码块标题null
    url跳转链接。若配置 link text ，则显示链接标题文字，点击后跳转null
    link text链接显示的标题文字link
    mark行高亮显示，格式为 mark: 行号，行号开始 - 行号结束，其他行号 。 例如 mark:1,4-7,10 ，将高亮显示第 1、4、5、6、7、10 行null
    command命令行提示符，格式为 command:("提示内容": 行号，行号 ||"提示内容": 行号开始 - 行号结束) <br /> 例如 command:("[root@localhost] $":1,9-10||"[admin@remotehost] #":4-6)null

  3. quiz 练习题。在文章头部参数区配置 quiz: true 可以设置练习题。使用 {.quiz} 等标签标注题型及选项答案。功能基于 markdown-it-bracketed-spans 和 markdown-it-attrs 。

    标签说明
    {.quiz}选择题
    {.quiz .multi}多选题
    {.quiz .true}正确的判断题
    {.quiz .false}错误的判断题
    {.quiz .fill}填空题
    []{.gap}空白下划线
    [答案内容]{.gap}答案内容带下划线
    {.options}ABCDE 选项
    {.correct}选择题的正确选项
    >答案解析
    [8.4]{.mistake}错题备注

  4. emoji 绘文字。 emoji 表情基于 markdown-it-emoji ，表情对应标签参考 emoji 表情。

  5. effects 文字特效。效果基于 markdown-it-ins 、 markdown-it-bracketed-spans 和 markdown-it-attrs 。

  6. spoiler 文字隐藏。样式效果基于 markdown-it-spoiler 和 markdown-it-attrs 。

    !! 黑幕黑幕黑幕黑幕黑幕黑幕！！： 光标悬浮于文字上显示内容
    ！！模糊模糊模糊模糊模糊模糊！！{.bulr} ： 光标选中文字显示内容

  9. tab 标签块。基于 markdown-it-container ，可以实现同行切换显示不同内容。
