# 线上部署

本地部署的博客站点只能在一台设备上浏览，要随时随地查看或者分享给其他人还需要将博客发布到线上。

线上部署博客可以通过架设自己的云服务器或者借助 Github、Gitee 平台等。因为云服务器的设备、带宽及流量等都是收费的，而 Gitee 虽然适合国内访问，但是更新内容时仍然需要手动部署，服务也时常关闭，所以体验并不好。

而 Github 平台的 Pages 服务可以免费托管 Hexo 框架搭建的个人网站，并且支持配置个人域名访问，所以很适合个人的博客分享。

密钥添加后，就可以尝试将本地博客发布到这个仓库实现线上部署了。

在博客文件夹下点击鼠标右键并打开 Git Bash Here 命令控制台。

首先输入以下命令安装 hexo-deployer-git 插件。

npm install hexo-deployer-git --save

之后重新解析生成博客静态资源，并使用 Hexo 的发布命令，将本地博客内容发布到远程仓库，Github Pages 服务会自动完成博客的线上部署。

最后在浏览器中打开 username.github.io 即可查看线上博客，部署需要一定时间，如很长时间未加载出页面，可以在仓库的 Actions 工作页面中查看是否部署成功。

# 个人域名

Github Pages 服务不仅提供了域名 username.github.io 进行访问，还可以绑定个人域名作为访问网址，方便网站的分类及搜索等。

个人域名可以在阿里云、华为云等域名服务商处购买，选择首年及续费比较便宜的可以控制维护成本。

有了域名之后，在博客目录的 source 文件夹下新建一个不带文件类型后缀的名称为 CNAME 的文件，在文件中写入个人域名，如 yunian.top 即可。 CNAME 文件可以由 .txt 类型的文本文件中写入个人域名后，重命名文件为 CNAME 并去除 .txt 后缀得到，写入的个人域名不要带有 www 及 http 等字样。

要让域名定向到博客网页，还需要进行域名解析。下面以阿里云域名配置为例：

打开域名控制台，点击左侧菜单中的 域名列表 。

点击需要配置的个人域名。

点击 域名解析

选择 添加记录

记录类型选择 CNAME ，一级域名配置主机记录为 www 即可，记录值填入 Github 提供的 username.github.io 域名，最后点击确定添加。

同理，再添加四条 “A” 类型记录，主机记录选择 @ ，记录值分别填写下面四个 IP 地址，用于指向 Github平台 服务器。

配置完成后需要等待一段时间进行解析，解析完成后使用个人域名就可以访问博客网站了。
