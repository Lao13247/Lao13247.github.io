# 线上部署

本地部署的博客站点只能在一台设备上浏览，要随时随地查看或者分享给其他人还需要将博客发布到线上。

线上部署博客可以通过架设自己的云服务器或者借助 Github、Gitee 平台等。因为云服务器的设备、带宽及流量等都是收费的，而 Gitee 虽然适合国内访问，但是更新内容时仍然需要手动部署，服务也时常关闭，所以体验并不好。

而 Github 平台的 Pages 服务可以免费托管 Hexo 框架搭建的个人网站，并且支持配置个人域名访问，所以很适合个人的博客分享。

使用 SSH 方式的好处是推送到远程仓库时无需账号登录，代码托管平台会验证本机所对应的公钥和私钥来识别推送方，所以现在先生成一下公钥和私钥。

打开路径为 C:\Users\用户名\.ssh 的目录，如没有这个文件夹可以新建一个。

保存密钥的文件名称默认为 id_rsa ，如果只配置单个 SSH 密钥，可以直接回车确认，使用默认文件名生成密钥。若有多个密钥的需求，文件名必须修改，要不然每次默认文件名生成的密钥仅会覆盖旧密钥。

之后是输入 SSH 密码以及确认密码，建议不设置，否则推送时可能还要输入密码，失去了 SSH 免登录的便捷。

到下面这个页面就是密钥成功生成了，可以打开 C:\Users\用户名\.ssh 目录查看，打开对应的文件名后缀为 .pub 的公钥文件，使用 Ctrl+A 快捷键选中全部内容进行复制，再到 Github平台 添加上公钥后，就可以使用 SSH 方式推送代码到仓库了。

登录 Github 平台，点击右上角的个人头像，选择 Settings 进入个人设置

选择 SSH and GPG keys 配置，

点击 New SSH key 添加密钥，

添加好密钥后，就可以尝试将本地的博客内容发布到这个仓库进行线上部署了。

在博客文件夹下点击鼠标右键并打开 Git Bash Here 命令控制台。

首先输入以下命令安装 hexo-deployer-git 插件

npm install hexo-deployer-git --save

之后重新解析生成博客静态资源，并使用 Hexo 的发布命令，将本地博客内容发布到远程仓库，Github Pages 服务会自动完成博客的线上部署。

最后在浏览器中打开 username.github.io 即可查看线上博客，部署需要一定时间，如很长时间未加载出页面，可以在仓库的 Actions 工作页面中查看是否部署成功。
